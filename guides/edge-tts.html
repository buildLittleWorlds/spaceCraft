Build Guide · API Wrapper Pattern

Edge TTS

1,000+ likes. Zero local models. This Space wraps Microsoft's free speech API in a Gradio interface. Here's how it works.

View the Space on Hugging Face

What This Space Does

You type text, pick a voice from dozens of options across many languages, and get high-quality synthesized speech back. The audio sounds professional — natural intonation, proper pacing, clear pronunciation. And it runs on free CPU because it doesn't run a model at all.

The Architecture

User types text → edge-tts library → Microsoft TTS servers → Audio file returned → gr.Audio plays it

This is the API wrapper pattern from Chapter 4. The Space is a frontend. All the speech synthesis happens on Microsoft's servers. The edge-tts Python library handles the HTTP communication.

Key insight: The edge-tts library doesn't need an API key. It wraps Microsoft Edge's online speech service, which is freely available. Your requirements.txt is just two lines: gradio and edge-tts. That's it. No torch, no transformers, no model frameworks.

The Code Pattern

Here's the essential structure of an Edge TTS Space. This isn't the exact source code — it's the pattern you'd use to build your own version:

1. Import and voice setup

import gradio as gr import edge_tts import asyncio import tempfile
Only four imports. edge_tts handles the API communication. asyncio is needed because edge-tts uses async/await. tempfile creates temporary audio files.

2. List available voices

async def get_voices(): voices = await edge_tts.list_voices() return { f"{v['ShortName']} - {v['Locale']}": v['ShortName'] for v in voices }
This calls Microsoft's service to get all available voices. There are dozens — English, Chinese, Spanish, Japanese, and more. Each voice has a short name like en-US-AriaNeural that you pass to the synthesis function.

3. The synthesis function

async def synthesize(text, voice_name): communicate = edge_tts.Communicate(text, voice_name) with tempfile.NamedTemporaryFile( suffix=".mp3", delete=False ) as tmp: await communicate.save(tmp.name) return tmp.name
This is the core function. edge_tts.Communicate() creates a synthesis request. .save() sends the text to Microsoft, receives the audio, and writes it to a temp file. Gradio's gr.Audio component can play this file directly.

4. Gradio interface

def tts(text, voice): return asyncio.run(synthesize(text, voice)) demo = gr.Interface( fn=tts, inputs=[ gr.Textbox(label="Text to speak", lines=3), gr.Dropdown( choices=list(voices.keys()), label="Voice", value="en-US-AriaNeural" ) ], outputs=gr.Audio(label="Generated speech"), title="Text to Speech", description="Type text, pick a voice, get audio." ) demo.launch()
A simple gr.Interface — text input + dropdown for voice selection → audio output. The asyncio.run() wrapper bridges Gradio's synchronous function calls with edge-tts's async API. The dropdown is populated with the voice list from step 2.

What You Could Change

  1. Add speed and pitch controls. edge_tts.Communicate() accepts rate and pitch parameters (e.g., rate="+20%", pitch="-10Hz"). Add gr.Slider components for these. This turns the Space into a parameter sweep tool.
  2. Filter voices by language. Instead of one giant dropdown, add a language selector that filters the voice list. Two dropdowns: language → voice. Better UX for users who know which language they want.
  3. Chain with a translation API. Add a step before TTS: translate the input text using the HF Inference API or LibreTranslate, then synthesize the translated text. Now you have a pipeline: type English → get Spanish audio.

The lesson: This Space has 1,000+ likes and runs zero ML inference. The value isn't in the model — it's in the interface design, the voice selection UX, and the fact that it wraps a powerful free service in a clean, shareable tool. When you build your own API-wrapper Space, that's where your effort should go: making the API easy and pleasant to use.
