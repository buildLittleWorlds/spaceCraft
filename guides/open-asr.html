Open ASR Leaderboard

1,200+ likes. Zero ML models. This Space is pure data visualization — and it's one of the most popular Spaces on Hugging Face.

View the Space on Hugging Face

What This Space Does

The Open ASR Leaderboard displays benchmark results for automatic speech recognition models. You can sort models by score, filter by language or dataset, search for specific models, and compare results side by side. It's a research tool used by the audio ML community to track which ASR models perform best on which tasks.

And it loads no models. It runs no inference. It doesn't use transformers or torch. It just displays data.

This is the most counterintuitive lesson in SpaceCraft: Some of the most liked, most used, most impactful Spaces on Hugging Face don't run ML at all. They curate, organize, and present data in ways that help people make decisions. That's a complete, valuable product — and it runs on free CPU because all it does is serve a table.

The Architecture

The architecture is almost comically simple compared to the other guides:

  1. Load benchmark data from a file (CSV, JSON, or a HF Dataset)
  2. Put it in a DataFrame
  3. Display it with Gradio's table component
  4. Add sorting, filtering, and search

That's the whole app. No model loading, no API calls, no async code. The value is in the data curation and the interface design.

The Code Pattern

1. Load data at startup

import pandas as pd # Load from a CSV in your repo df = pd.read_csv("results.csv") # Or load from a Hugging Face Dataset from datasets import load_dataset ds = load_dataset("hf-audio/asr-benchmarks") df = ds["train"].to_pandas()
Data can come from a CSV file committed to the repo, a JSON file, or a Hugging Face Dataset (which is like a CSV but hosted on HF with versioning). For a student project, a CSV in the repo is simplest. The Open ASR Leaderboard likely uses an HF Dataset so the data can be updated independently of the Space code.

2. Display with gr.Dataframe

import gradio as gr with gr.Blocks() as demo: gr.Markdown("# Open ASR Leaderboard") gr.Markdown("Compare speech recognition models " "across multiple benchmarks.") gr.Dataframe( value=df, interactive=False, wrap=True )
gr.Dataframe renders a sortable table. Click any column header to sort. Set interactive=False so users can't edit the data, only view it. That's already a functional leaderboard in about 10 lines of code.

3. Add filtering (basic approach)

def filter_data(search_text, language): filtered = df.copy() if search_text: mask = df.apply( lambda row: search_text.lower() in row.to_string().lower(), axis=1 ) filtered = filtered[mask] if language and language != "All": filtered = filtered[ filtered["language"] == language ] return filtered with gr.Blocks() as demo: gr.Markdown("# Leaderboard") with gr.Row(): search = gr.Textbox( label="Search", placeholder="Filter models..." ) lang = gr.Dropdown( choices=["All"] + sorted( df["language"].unique().tolist() ), label="Language", value="All" ) table = gr.Dataframe(value=df, interactive=False) search.change(filter_data, [search, lang], table) lang.change(filter_data, [search, lang], table) demo.launch()
This is the full pattern: text search + dropdown filter, both triggering the same filter function that updates the table. The .change() event means the table updates live as the user types or selects. For a leaderboard with more filter options, add more dropdowns/checkboxes and more columns to the filter function.

4. Advanced: gradio-leaderboard component

from gradio_leaderboard import Leaderboard # Drop-in replacement for gr.Dataframe with # built-in search, column filters, and sorting Leaderboard( value=df, search_columns=["model_name"], filter_columns=["language", "dataset"], )
The gradio-leaderboard package is a third-party Gradio component specifically built for leaderboard Spaces. It adds search, column-specific filters, and sorting controls with minimal code. Install it with pip install gradio-leaderboard. Many of the popular leaderboard Spaces on HF use this component.

Where the Data Comes From

The hardest part of building a leaderboard isn't the code — it's the data. You need benchmark results, and those need to be accurate and up-to-date. Common approaches:

  • Manual curation: You run benchmarks yourself and commit the results as a CSV. SpaceCraft itself works this way — the leaderboard data is hand-curated.
  • HF Dataset: Store results in a Hugging Face Dataset. This lets you update data without redeploying the Space, and others can submit results via pull requests.
  • Submission system: The most advanced leaderboards let users submit their own models for evaluation. The Open ASR Leaderboard does this — researchers submit models, benchmarks run automatically, and results appear on the leaderboard.

For a student project, manual curation (a CSV in the repo) is the right starting point. You can always upgrade later.

What You Could Change

  1. Build a leaderboard for your own domain. Curate benchmark results for a topic you care about: best small language models under 1B parameters, best OCR tools for handwritten text, best CPU-friendly TTS voices. The code pattern is the same — only the data changes.
  2. Add charts. Gradio has a gr.Plot component that renders Matplotlib or Plotly charts. Add a bar chart showing the top 10 models, or a scatter plot comparing two metrics. Visual summaries help users spot trends faster than scanning a table.
  3. Combine with an API. Fetch live data from a free API instead of a static CSV. A "trending models" leaderboard could pull data from the HF API. A "news sentiment" dashboard could pull from GNews and classify on the fly (that's now a pipeline).

Connection to the Course

  • Session 9 (User-Centered Design): The Open ASR Leaderboard scores high on Craft because every design choice serves its audience: ML researchers comparing models. The filters, column headers, and sort options are exactly what that audience needs. Designing a leaderboard for a different audience would mean different columns and different filters — that's user-centered design.
  • Session 10 (Build From Scratch): A data dashboard is one of the three starter patterns in Chapter 5. If your final project is a leaderboard or comparison tool, this guide shows you how the pattern scales from 10 rows to 1,000+.
  • Comparative Analysis: A leaderboard IS comparative analysis, institutionalized. It makes comparison the default interaction. When you build a leaderboard, you're building a tool that helps others do comparative analysis on your curated data.

The lesson: You don't need a model to build something people love. The Open ASR Leaderboard is one of the most popular Spaces on Hugging Face because it solves a real problem — comparing ASR models is tedious, and this Space makes it easy. The code is simple. The data curation is the hard part. If you can curate useful data and present it clearly, you have a Space that matters.
