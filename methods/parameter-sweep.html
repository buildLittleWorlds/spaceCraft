Parameter Sweep

Change one setting at a time. Hold everything else constant. That's experimental design.

What It Is

A parameter sweep means systematically changing one variable while keeping everything else the same, then observing how the output changes. This is the foundation of experimental design — isolating variables so you know what actually caused a difference. If you change two things at once, you don't know which one mattered.

Every time a developer tunes a model's temperature setting, they're doing a parameter sweep. Every time a team A/B tests a product feature, they're isolating a variable. This method is everywhere.

When You Use It

Use a parameter sweep when a Space or model has adjustable settings and you want to understand what each one does. Voice selection, speed, temperature, top-p, max length, number of results — any slider, dropdown, or number field is a variable you can sweep.

The Steps

  1. Pick one variable. Temperature, voice, language, max results — whatever the Space lets you control.
  2. Lock everything else. Same model, same input text, same values for all other settings. Write down your locked values.
  3. Sweep the variable. Go from the lowest setting to the highest in regular steps. If it's temperature: try 0.1, 0.3, 0.5, 0.7, 0.9, 1.2, 1.5.
  4. Record the output at each step. Does it change gradually or suddenly? Is there a sweet spot? What happens at the extremes?
  5. Write a "settings recipe." Document which combinations work best for which tasks. "For formal business writing: temperature 0.3, top-p 0.9. For creative brainstorming: temperature 0.9, top-p 0.95."

Kokoro TTS Text-To-Speech Unlimited

What to try: Type the same sentence into a TTS Space. Keep the text identical. Change only the voice. Listen to each version. Which voices sound more natural for this particular sentence? Now lock the voice and change the speed (if available). Does faster speech change the perceived emotion? You're doing a parameter sweep — isolating one variable at a time and observing the effect.

What You'll Discover

The key insight is that the same model can produce dramatically different output depending on settings — and those settings are the same controls that developers tune on ChatGPT, Claude, and every production AI system. The only difference is scale. When you adjust temperature on a small text generator, you're turning the same knob that OpenAI turns on GPT-4. Understanding what that knob does at small scale means you'll understand it at any scale.

Try it yourself: Open any TTS Space from the leaderboard. Type a single sentence. Try every available voice with the same sentence. Rank them from "most natural" to "least natural" for that sentence. Then try a completely different sentence — does your ranking change? If so, the "best" voice depends on the content, not just the voice itself. That's a finding worth documenting.
