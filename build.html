Chapter 5

Build Your First Space

From zero to a live, deployed application. Three paths depending on what you want to build.

Before You Start

You need a free Hugging Face account. That's it. No local Python installation, no terminal, no git knowledge. You can build and deploy a Space entirely through the Hugging Face web interface.

Creating a Space

1 Go to huggingface.co/new-space.

2 Give your Space a name. Keep it short and descriptive: mood-meter, headline-finder, text-translator.

3 Choose Gradio as the SDK.

4 Leave the hardware as CPU Basic (Free).

5 Click Create Space.

You now have a live URL. The Space is empty, but it exists. Time to add code.

Three Paths

Below are three complete Spaces you can build right now. Each one teaches a different CPU strategy from the earlier chapters. Pick the one that interests you most.

Path A: The Classifier (Local Model)

This Space runs a small sentiment analysis model directly on CPU. It demonstrates the specialized model strategy from Chapter 3.

app.py

import gradio as gr from transformers import pipeline classifier = pipeline( "sentiment-analysis", model="distilbert-base-uncased-finetuned-sst-2-english" ) def analyze(text): if not text.strip(): return "Please enter some text." result = classifier(text)[0] label = result["label"] score = round(result["score"] * 100, 1) return f"**{label}** ({score}% confidence)" demo = gr.Interface( fn=analyze, inputs=gr.Textbox( label="Enter text", placeholder="Type a sentence to analyze...", lines=3 ), outputs=gr.Markdown(label="Result"), title="Mood Meter", description="Analyze the sentiment of any text. " "Uses DistilBERT (67M parameters) running on free CPU." ) demo.launch()

requirements.txt

transformers torch

Why this works on CPU: DistilBERT has 67 million parameters — well under the 1B threshold from Chapter 3. It loads in about 250 MB of RAM and classifies text in under a second on CPU. It's a perfect example of a small, specialized model.

Path B: The API Tool (Free API)

This Space calls an external API to get data, then displays it. No ML model at all. It demonstrates the API wrapper pattern from Chapter 4.

app.py

import gradio as gr import requests def lookup_word(word): if not word.strip(): return "Please enter a word." url = f"https://api.dictionaryapi.dev/api/v2/entries/en/{word.strip()}" try: response = requests.get(url, timeout=10) if response.status_code == 404: return f"**{word}** not found in the dictionary." data = response.json()[0] except Exception: return "Something went wrong. Please try again." output = f"# {data['word']}\n\n" for meaning in data.get("meanings", []): part = meaning.get("partOfSpeech", "") output += f"### {part}\n\n" for defn in meaning.get("definitions", [])[:3]: output += f"- {defn['definition']}\n" if defn.get("example"): output += f" *Example: {defn['example']}*\n" output += "\n" return output demo = gr.Interface( fn=lookup_word, inputs=gr.Textbox(label="Enter a word", placeholder="e.g. serendipity"), outputs=gr.Markdown(label="Definition"), title="Dictionary Lookup", description="Look up any English word. " "Uses the free Dictionary API — no API key needed." ) demo.launch()

requirements.txt

gradio requests

Why this is great for learning: No ML model, no API key, no secrets to manage. The Dictionary API is completely free and keyless. You can go from empty Space to working tool in five minutes. And it's genuinely useful — people actually look up words.

Path C: The Dashboard (No Inference)

This Space displays data in an interactive table. No model, no API — just data visualization. It demonstrates the no-inference pattern from Chapter 3.

app.py

import gradio as gr import pandas as pd # Data could come from a CSV file in your repo data = { "Model": [ "DistilBERT", "BERT-base", "RoBERTa-base", "ALBERT-base", "TinyBERT", "MobileBERT" ], "Parameters": [ "67M", "110M", "125M", "12M", "15M", "25M" ], "Task": [ "Sentiment", "NER", "Classification", "QA", "Distillation", "Mobile NLP" ], "CPU Friendly": [ "Yes", "Yes", "Yes", "Very", "Very", "Very" ], "Notes": [ "Most popular small model", "The original transformer", "Better training procedure", "Parameter sharing = tiny", "Distilled from BERT", "Designed for phones" ] } df = pd.DataFrame(data) def search_models(query): if not query.strip(): return df mask = df.apply( lambda row: query.lower() in row.to_string().lower(), axis=1 ) return df[mask] demo = gr.Interface( fn=search_models, inputs=gr.Textbox( label="Search", placeholder="Filter models (e.g. 'tiny', 'sentiment')..." ), outputs=gr.Dataframe(label="CPU-Friendly Models"), title="Small Model Explorer", description="Browse models that run well on free CPU. " "All under 130M parameters." ) demo.launch()

requirements.txt

gradio pandas

This pattern scales. Replace the hardcoded data with a CSV file in your repo, and you have a leaderboard. Add more columns, sorting, and filtering, and you're building something like the Open ASR Leaderboard (1,200+ likes on the SpaceCraft leaderboard).

Deploying Your Code

Once you've written your files, you need to get them into your Space. Two options:

Option 1: The Web Editor

On your Space page, click Files, then Add file → Create a new file. Paste your code, name the file app.py, and commit. Do the same for requirements.txt. Your Space will start building automatically.

Option 2: Git

If you're comfortable with git, clone your Space repo and push files the normal way:

git clone https://huggingface.co/spaces/YOUR-USERNAME/YOUR-SPACE cd YOUR-SPACE # add your app.py and requirements.txt git add . git commit -m "Initial app" git push

What Happens After You Push

  1. Hugging Face detects the new commit
  2. It installs your dependencies from requirements.txt
  3. It runs your app.py
  4. Your Space goes live at its URL

First build takes a few minutes (installing packages). After that, updates are faster. If the build fails, check the Logs tab on your Space page — it shows exactly what went wrong.

Expect errors. Your first deploy will probably fail. A misspelled package name, a missing import, a function that doesn't return the right type. This is normal. Read the error log, fix the issue, push again. Every Space on the leaderboard went through this cycle.

After Your First Space

Once you have something working:

  • Share the URL. Every Space has a public link. Show it to someone.
  • Add it to a Collection. On Hugging Face, create a Collection and add your Space to it. This is the start of your portfolio.
  • Study the next level. Browse the leaderboard and pick a Space that does something similar to yours but better. Read its code. What did they do differently? Chapter 6 teaches you how to learn from other people's Spaces systematically.
  • Iterate. The best Spaces weren't built in one pass. They were built, tested, shown to someone, revised, and polished. Your first version doesn't have to be your last.
