Chapter 9

Hugging Face as an API

Spaces are one way to use Hugging Face. But the whole platform is an API you can call from anywhere.

The Bigger Picture

Every chapter so far has focused on building Spaces — web apps deployed on Hugging Face. But Spaces are just one part of the platform. Hugging Face also gives you:

  • The Inference API — Run any public model on HF's servers from any code, anywhere
  • The huggingface_hub library — Search, download, and manage models and datasets programmatically
  • MCP servers — Spaces that other AI tools can call as services

Understanding these expands what you can build far beyond web apps.

The Inference API

You've already seen the Inference API in Chapter 4 — calling it from inside a Space. But you don't need a Space to use it. Any Python script, Jupyter notebook, or application can call the Inference API directly.

import requests API_URL = "https://api-inference.huggingface.co/models/distilbert-base-uncased-finetuned-sst-2-english" headers = {"Authorization": "Bearer YOUR_HF_TOKEN"} response = requests.post( API_URL, headers=headers, json={"inputs": "I love this movie!"} ) print(response.json()) # [{'label': 'POSITIVE', 'score': 0.9998}]

That's it. One HTTP request. The model runs on HF's servers (potentially on GPU), and you get the result back. You can run this from a Google Colab notebook, a local Python script, a JavaScript app, or a command-line tool.

What this means for you: You can build things that aren't Spaces. A Python script that analyzes a folder of documents. A notebook that classifies hundreds of rows in a spreadsheet. A Discord bot that responds to messages using an HF model. Spaces are for building web apps. The Inference API is for building anything.

The Inference API works with most public models on HF. Swap the model URL to change the task:

Every model page on Hugging Face has a "Deploy" button that shows you the exact API call for that model.

Rate Limits

The free tier is rate-limited — you get a burst of requests per minute, and models that aren't loaded may take 20-30 seconds on the first call. For student projects and prototyping, this is fine. For production use, there are paid Inference Endpoints with dedicated hardware and no rate limits.

The huggingface_hub Library

The huggingface_hub Python library gives you programmatic access to the entire Hub — models, datasets, Spaces, and more.

pip install huggingface_hub

Search for models

from huggingface_hub import HfApi api = HfApi() # Find the most-downloaded sentiment models models = api.list_models( task="text-classification", sort="downloads", direction=-1, limit=5 ) for m in models: print(f"{m.id} — {m.downloads:,} downloads")

This is how you can programmatically discover models instead of browsing the website. Filter by task, language, library, size — anything you can filter on the web UI, you can filter in code.

Download a model

from huggingface_hub import snapshot_download # Download a model to use locally path = snapshot_download("distilbert-base-uncased-finetuned-sst-2-english")

Run inference with InferenceClient

from huggingface_hub import InferenceClient client = InferenceClient(token="YOUR_HF_TOKEN") # Text classification result = client.text_classification( "I love this movie!", model="distilbert-base-uncased-finetuned-sst-2-english" ) # Translation translated = client.translation( "Hello, how are you?", model="Helsinki-NLP/opus-mt-en-fr" ) # Summarization summary = client.summarization( "Long article text here...", model="facebook/bart-large-cnn" )

InferenceClient is a cleaner way to call the Inference API than raw requests.post(). It handles authentication, serialization, and error messages for you. Each task has its own method with typed parameters.

When to use InferenceClient vs. raw requests: Use InferenceClient when you're writing Python code and want clean, typed API access. Use raw requests when you need to understand exactly what's being sent (good for learning) or when you're not using Python.

Inference API vs. Local Inference

You now have two ways to run a model: load it locally (like in Chapter 5, Path A) or call the Inference API. Each has tradeoffs:

Local (in your Space)Inference API
SpeedInstant after loading1-3 seconds per call
Model sizeMust fit in CPU RAMAny size (runs on HF's GPUs)
Offline?Works after model loadsNeeds internet
Rate limitsNoneYes (free tier)
PrivacyData stays localData sent to HF servers
Best forSmall models, fast responseLarge models, occasional use

Rule of thumb: If the model is under 500M parameters and you need fast responses, load it locally. If the model is large or you only call it occasionally, use the Inference API. For pipelines, you can mix both — a local classifier for fast text processing and an API call for a larger model when needed.

MCP Servers on Spaces

Some Spaces on the leaderboard are tagged as MCP (Model Context Protocol) servers. This means they can be called as tools by AI assistants like Claude. Instead of a human using the Gradio interface, an AI agent sends requests to the Space programmatically.

Why this matters: MCP turns a Space from a human-facing tool into a machine-facing service. A Space that does OCR can become a tool that Claude uses to read documents. A Space that does translation can become a service that any AI workflow calls when it needs to translate text. Building a Space that works as an MCP server means your tool can be used by both humans and AI agents.

On the SpaceCraft leaderboard, core OCR, Chatterbox TTS, and CLIP Interrogator are all MCP-enabled. They work as normal Gradio Spaces for human users and simultaneously serve as MCP tools for AI agents.

Building Beyond Spaces

With the Inference API and huggingface_hub, here are things you can build that aren't Spaces:

  • A Colab notebook that processes a CSV file — classify every row, translate every entry, summarize every document — using Inference API calls. No Space needed.
  • A command-line tool that takes a file path, runs it through an HF model, and outputs results. A Python script you run locally.
  • A Discord or Slack bot that responds to messages using HF models. The bot runs on your server (or a free hosting service) and calls the Inference API when someone @mentions it.
  • A batch processor that downloads models locally with snapshot_download() and processes thousands of items without rate limits or network latency.

Spaces are still the easiest way to share something publicly. But knowing how to use HF as an API means you're not limited to building web interfaces. You can integrate HF models into any workflow.

What You Should Know Now

After this chapter, you understand that Hugging Face is a platform with three layers:

  1. The Hub — Models, datasets, and Spaces. Browse, search, and curate.
  2. The Inference API — Run any model from any code, anywhere. Free tier for prototyping.
  3. Spaces — Deploy web apps that use models, APIs, or data. Free CPU tier.

Everything in this textbook has been about layer 3 (Spaces). Now you know layers 1 and 2 exist, and how to use them. That opens up a much wider world of things you can build.
